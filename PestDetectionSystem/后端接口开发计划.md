## 二、文章相关接口（待开发）

> 依赖数据库表：`article`、`article_like`、`article_comment`、`comment_like`

### 2.1 文章点赞 🔴

```http
POST /api/article/{articleId}/like
Authorization: Bearer <token>
```

**返回值**

```json
{
  "code": 200, "message": "success",
  "data": { "articleId": 123456789, "liked": true, "likeCount": 24 }
}
```

**实现步骤**
1. 查询 `article_like` 表判断是否已点赞
2. 已点赞则删除记录（取消），否则插入新记录
3. 更新 `article.like_count`
4. 返回当前点赞状态和总数

---

### 2.2 文章评论列表 🔴

```http
GET /api/article/{articleId}/comments?page=1&pageSize=20&sortBy=newest
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `page` | `int` | 否 | 页码，默认 1 |
| `pageSize` | `int` | 否 | 每页数量，默认 20，最大 100 |
| `sortBy` | `string` | 否 | `newest`（最新）\| `hottest`（最热） |

**返回值**

```json
{
  "code": 200, "message": "success",
  "data": {
    "total": 150, "pageNum": 1, "pageSize": 20, "pages": 8,
    "list": [
      {
        "id": 987654321, "userId": 123456, "username": "农民张三",
        "userImage": "data:image/jpeg;base64,...",
        "content": "这篇文章写得很好！", "likeCount": 5,
        "replyCount": 2, "isLiked": false, "createdTime": 1704067200000,
        "replies": [
          {
            "id": 987654322, "userId": 789012, "username": "博主李四",
            "userImage": "data:image/jpeg;base64,...",
            "content": "@农民张三 感谢反馈！", "likeCount": 1,
            "isLiked": false, "createdTime": 1704070800000
          }
        ]
      }
    ]
  }
}
```

**实现步骤**
1. 分页查询顶级评论（`parent_id IS NULL`）
2. 每条评论查询最多 3 条最新回复
3. 从 `comment_like` 判断当前用户是否已点赞各评论

---

### 2.3 发布评论 🔴

```http
POST /api/article/{articleId}/comments
Authorization: Bearer <token>
Content-Type: application/json
```

**请求参数**

```json
{ "content": "这篇文章对我很有启发！", "parentId": null }
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `content` | `string` | 是 | 评论内容，1-1000 字 |
| `parentId` | `long` | 否 | 父评论 ID，null 表示顶级评论 |

**返回值**

```json
{
  "code": 200, "message": "success",
  "data": {
    "id": 987654325, "articleId": 123456789, "userId": 123456,
    "username": "当前用户", "userImage": "data:image/jpeg;base64,...",
    "content": "这篇文章对我很有启发！", "parentId": null,
    "likeCount": 0, "replyCount": 0, "createdTime": 1704153600000
  }
}
```

**实现步骤**
1. 验证评论内容长度（1-1000 字）
2. 若 `parentId` 非空，验证父评论存在且未删除
3. 插入评论记录（`status=1` 直接发布）
4. 若有 `parentId`，更新父评论的 `reply_count`

---


**返回值**

---

### 2.5 删除评论 🟡

```http
DELETE /api/comment/{commentId}
Authorization: Bearer <token>
```

- 只有评论作者或管理员可删除
- 级联删除子评论，更新父评论 `reply_count`

**返回值**

```json
{ "code": 200, "message": "评论已删除", "data": null }
```

## 三、预警相关接口（待开发）

> 依赖数据库表：`warning`、`warning_read`

### 3.1 新增预警 🔴（管理员）

```http
POST /api/warning
Authorization: Bearer <token>
Content-Type: application/json
```

**请求参数**

```json
{
  "title": "小麦赤霉病预警", "content": "受近期降水影响...",
  "region": "河南省", "pestName": "小麦赤霉病",
  "severity": 3, "status": 1
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | `string` | 是 | 预警标题，1-200 字 |
| `content` | `string` | 是 | 预警内容 |
| `region` | `string` | 否 | 受影响地区 |
| `pestName` | `string` | 否 | 关联病虫害名称 |
| `severity` | `int` | 是 | 1=低，2=中，3=高 |
| `status` | `int` | 是 | 0=下架，1=上架 |

---

```json
        "status": 1, "publishTime": 1704240000000, "publisherId": 999,
```

---

### 3.3 标记预警已读 🔴

```http
POST /api/warning/{warningId}/read
Authorization: Bearer <token>
```

**返回值**

```json
{
  "code": 200, "message": "success",
  "data": { "warningId": 555666777, "userId": 123456, "readTime": 1704240600000 }
}
```

**实现步骤**
1. 检查预警存在且已上架
2. 使用 `INSERT IGNORE` 防止重复记录
3. 返回已读时间

---


**返回值**

> 返回最近 3 条未读预警摘要，供消息中心显示小红点/badge

### 3.5 获取预警详情 🟡

```http
GET /api/warning/{warningId}
Authorization: Bearer <token>
```

**实现步骤**
1. 查询预警详情
2. 查询当前用户是否已读
3. `viewCount` 自动 +1
4. 若未读则自动标记已读

## 五、技术实现建议

- **ORM**：MyBatis Plus（与现有代码保持一致）
- **分页**：PageHelper
- **ID 生成**：雪花 ID（与现有模块保持一致）
- **缓存**：Redis（热门文章、预警列表）

### 关键业务逻辑
1. `article_like`、`comment_like`、`warning_read` 表需 UNIQUE 索引防重复
2. 点赞/取消点赞时同步更新对应 `like_count` 字段
3. 删除评论时级联删除子评论和点赞记录
4. `viewCount` 可用 MQ 异步更新提高响应速度

### 安全考虑
1. 管理员预警接口需校验 `role=0`
2. 评论内容做关键词过滤
3. 评论、点赞接口做单用户请求频率限制
4. 使用 MyBatis Plus 参数化查询，禁止手写拼接 SQL
