# 后端接口文档

## 1. 基本信息

- 服务端口：`8888`
- 统一前缀：`/api`
- 完整前缀示例：`http://localhost:8888/api`

## 2. 统一返回格式

大部分接口返回统一结构 `Result<T>`：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

部分成功接口会返回业务提示语：

```json
{
  "code": 200,
  "message": "注册成功",
  "data": null
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | `int` | 状态码，成功通常为 `200` |
| `message` | `string` | 提示信息 |
| `data` | `any` | 业务数据 |

## 3. 认证说明

需要登录的接口必须在请求头中携带 JWT：

```http
Authorization: Bearer <token>
```

### 权限规则

#### 普通用户接口

以下接口需要用户 token，且角色为 `1`：

- `/user/bind`
- `/user/info`
- `/user/username`
- `/user/sex`
- `/user/avatar`
- `/detect`
- `/detect/records`
- `/chat/session`
- `/chat/sessions`
- `/chat/session/{sessionId}/messages`
- `/chat/send`
- `/chat/session/{sessionId}`
- `/chat/quota`

以下接口无需登录：

- `/user/login`
- `/user/register`
- `/llm/test`
- `/health`

#### 管理员接口

以下接口需要管理员 token，且角色为 `0`：

- `/admin/allusers`

### 认证失败的实际表现

拦截器直接返回 HTTP 状态码和纯文本，不走 `Result<T>` 包装：

- 未携带或格式错误：`401`，响应文本 `Bearer token is required`
- token 过期：`401`，响应文本 `Token out of date`
- token 无效：`401`，响应文本 `Invalid Token`
- 角色不匹配：`403`，响应文本 `Access denied`

## 4. 用户模块接口

### 4.1 用户登录

- 功能：用户使用账号和密码登录，返回用户信息和 JWT
- 请求方式：`POST`
- 调用路径：`/api/user/login`

#### 请求参数

`Content-Type: application/json`

```json
{
  "account": "user@example.com",
  "password": "123456"
}
```

#### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `account` | `string` | 是 | 登录账号，代码按手机号或邮箱查询 |
| `password` | `string` | 是 | 登录密码 |

#### 返回值

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 123456789,
    "role": 1,
    "username": "test_user",
    "email": "user@example.com",
    "phone": "13800138000",
    "sex": 1,
    "image": "data:image/jpeg;base64,...",
    "status": 1,
    "flag": 0,
    "token": "xxxxx.yyyyy.zzzzz"
  }
}
```

### 4.2 用户注册

- 功能：注册新用户
- 请求方式：`POST`
- 调用路径：`/api/user/register`

#### 请求参数

`Content-Type: application/json`

```json
{
  "username": "new_user",
  "password": "123456",
  "email": "new@example.com",
  "phone": "13800138000"
}
```

#### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `username` | `string` | 否 | 用户名，空时后端自动生成 `uid_用户ID` |
| `password` | `string` | 是 | 用户密码 |
| `email` | `string` | 否 | 邮箱 |
| `phone` | `string` | 否 | 手机号 |

#### 说明

- `email` 和 `phone` 至少填写一个
- 邮箱和手机号都会校验唯一性

#### 返回值

```json
{
  "code": 200,
  "message": "注册成功",
  "data": null
}
```

### 4.3 绑定手机号或邮箱

- 功能：给当前登录用户绑定手机号、邮箱，或同时绑定
- 请求方式：`PATCH`
- 调用路径：`/api/user/bind`
- 是否鉴权：是

#### 请求头

```http
Authorization: Bearer <token>
Content-Type: application/json
```

#### 请求参数

```json
{
  "phone": "13800138000",
  "email": "user@example.com",
  "bindType": "BOTH"
}
```

#### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `phone` | `string` | 否 | 绑定手机号 |
| `email` | `string` | 否 | 绑定邮箱 |
| `bindType` | `string` | 是 | `PHONE`、`EMAIL`、`BOTH`，大小写不敏感 |
| `id` | `long` | 否 | 前端无需传，后端会使用当前登录用户 ID 覆盖 |

#### 返回值

```json
{
  "code": 200,
  "message": "绑定成功",
  "data": null
```

### 4.7 上传头像

- 功能：上传并更新当前登录用户头像
- 请求方式：`POST`
- 调用路径：`/api/user/avatar`
- 是否鉴权：是
- `Content-Type`：`multipart/form-data`

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `file` | `File` | 是 | 图片文件，后端按 `image/*` 校验，大小不能超过 5MB |

#### 返回值

```json
{
  "code": 200,
  "message": "头像更新成功",
  "data": null
}
```

#### 业务错误码

| 错误码 | 说明 |
|---|---|
| `40050` | 头像文件不能为空 |
| `40051` | 只能上传图片文件 |
| `40052` | 图片大小不能超过 5MB |

#### 说明

- 头像保存目录来自 `application.yml` 中的 `user.base-path`
- 返回用户信息时，头像会被读取并转成 Base64 Data URI

## 6. 病虫害识别模块接口

### 6.1 病虫害识别

- 功能：上传图片 Base64，调用模型服务进行识别，并保存识别记录
- 请求方式：`POST`
- 调用路径：`/api/detect`
- 是否鉴权：是，用户权限

#### 请求参数

`Content-Type: application/json`

```json
{
  "imageBase64": "/9j/4AAQSkZJRgABAQ..."
}
```

#### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `imageBase64` | `string` | 是 | 图片 Base64 内容，不带 `data:image/...;base64,` 前缀 |

#### 返回值

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1743840000000,
    "userId": 123456789,
    "imageName": "unknown",
    "topLabel": "brown_spot",
    "confidence": 0.98,
    "createdTime": 1743840000000
  }
}
```

#### 说明

- 后端会先把图片保存到本地目录 `PestDetectionSystem/pest-images`
- 调用 Flask 模型服务时会自动补上 `data:image/jpeg;base64,` 前缀
- 实际调用的是模型服务的 `/predict`
- 识别结果置信度阈值为 `0.7`
- 即使置信度低于 `0.7`，当前实现仍会返回识别结果，但该记录 `status=0`，不会出现在历史记录接口中

### 6.2 获取识别记录

## 7. LLM 接口

### 7.1 简单问答

- 功能：调用大模型进行单轮问答
- 请求方式：`POST`
- 调用路径：`/api/llm/test`
- 是否鉴权：否

#### 请求参数

通过查询参数传递：

```http
POST /api/llm/test?message=水稻常见病虫害有哪些
```

#### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `message` | `string` | 是 | 用户输入的问题 |

#### 返回值

返回体结构取决于第三方 LLM 响应，当前后端直接将其包装进 `Result.data`：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "chatcmpl-xxx",
    "choices": [
      {
        "message": {
          "role": "assistant",
          "content": "常见病虫害包括..."
        }
      }
    ],
    "created": 1710000000,
    "model": "MiniMax-M2.7",
    "object": "chat.completion",
    "usage": {
      "prompt_tokens": 100,
      "completion_tokens": 50,
      "total_tokens": 150
    },
    "base_resp": {
      "status_code": 0,
      "status_msg": "success"
    }
  }
}
```

#### 说明

- 系统固定注入一条 system prompt，要求以农业病虫害专家助手身份进行中文回答
- 如果 `application.yml` 未配置 `llm.api-key`，调用会失败

### 8.4 发送消息

- 功能：发送消息并获得 LLM 回复
- 请求方式：`POST`
- 调用路径：`/api/chat/send`
- 是否鉴权：是

#### 请求参数

`Content-Type: application/json`

```json
{
  "sessionId": 1,
  "message": "水稻叶片发黄是什么原因？",
  "detectionId": 1001
}
```

#### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `sessionId` | `long` | 是 | 会话 ID，必须是当前用户已存在的会话 |
| `message` | `string` | 是 | 消息内容 |
| `detectionId` | `long` | 否 | 当前实现已定义字段，但服务逻辑未使用 |

#### 返回值

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "sessionId": 1,
    "answer": "可能原因有缺氮、病害或水分问题。",
    "promptTokens": 120,
    "completionTokens": 60,
    "totalTokens": 180
  }
}
```

#### 说明

- `sessionId` 不能为空，当前实现不会自动创建新会话
- 每次发送会先保存用户消息，再保存模型回复
- 首轮对话成功后，如果会话原本没有消息，会自动用用户首条消息前 20 个字符生成标题
- 调用 LLM 时会携带最多最近 20 条成功消息作为上下文
- 每个用户默认每日最多请求 `50` 次，可通过 `chat.quota.daily-max-requests` 配置覆盖

## 11. 本次对齐后的关键差异

1. 文档原先遗漏了病虫害识别模块，现已补充 `/detect` 和 `/detect/records`
2. `/detect/**` 实际需要用户 token，现已补充到鉴权说明
3. `/chat/send` 的 `sessionId` 实际必须传入，不会自动创建会话
