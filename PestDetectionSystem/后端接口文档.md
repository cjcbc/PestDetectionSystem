# 后端接口文档

## 1. 基本信息

- 服务端口：`8888`
- 统一前缀：`/api`
- 完整前缀示例：`http://localhost:8888/api`

## 2. 统一返回格式

大部分接口返回统一结构 `Result<T>`：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | `int` | 状态码，成功通常为 `200` |
| `message` | `string` | 提示信息 |
| `data` | `any` | 业务数据 |

## 3. 认证说明

需要登录的接口必须在请求头中携带 JWT：

```http
Authorization: Bearer <token>
```

### 权限规则

#### 普通用户接口（role 0 或 1）

- `/user/bind`、`/user/info`、`/user/username`、`/user/sex`、`/user/avatar`、`/user/password`、`/user/logout`
- `/detect`、`/detect/records`
- `/chat/session`、`/chat/sessions`、`/chat/session/{sessionId}/messages`、`/chat/send`、`/chat/send/stream`、`/chat/session/{sessionId}`、`/chat/quota`

#### 无需登录

- `/user/login`、`/user/register`
- `/llm/test`
- `/health`

#### 管理员接口（role 0 only）

- `/admin/**`（除 `/admin/login`、`/admin/logout`）

### 认证失败的实际表现

拦截器直接返回 HTTP 状态码和纯文本，不走 `Result<T>` 包装：

- 未携带或格式错误：`401`，响应文本 `Bearer token is required`
- token 过期：`401`，响应文本 `Token out of date`
- token 无效：`401`，响应文本 `Invalid Token`
- 角色不匹配：`403`，响应文本 `Access denied`

## 4. 用户模块接口

### 4.1 用户登录

- 请求方式：`POST /api/user/login`
- 是否鉴权：否

#### 请求参数

```json
{ "account": "user@example.com", "password": "123456" }
```

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `account` | `string` | 是 | 手机号或邮箱 |
| `password` | `string` | 是 | 登录密码 |

#### 返回值

```json
{
  "code": 200, "message": "success",
  "data": {
    "id": "123456789", "role": 1, "username": "test_user",
    "email": "user@example.com", "phone": "13800138000",
    "sex": 1, "image": "data:image/jpeg;base64,...",
    "status": 0, "flag": 0, "token": "xxxxx.yyyyy.zzzzz"
  }
}
```

> `status`：0=正常，1=锁定；`flag`：0=普通，1=达人，2=VIP；`sex`：0=男，1=女，2=未知

---

### 4.2 用户注册

- 请求方式：`POST /api/user/register`
- 是否鉴权：否

#### 请求参数

```json
{ "username": "new_user", "password": "123456", "email": "new@example.com", "phone": "13800138000" }
```

- `email` 和 `phone` 至少填写一个，均会校验唯一性
- `username` 为空时后端自动生成 `uid_用户ID`

#### 返回值

```json
{ "code": 200, "message": "注册成功", "data": null }
```

---

### 4.3 绑定手机号或邮箱

- 请求方式：`PATCH /api/user/bind`
- 是否鉴权：是

#### 请求参数

```json
{ "phone": "13800138000", "email": "user@example.com", "bindType": "BOTH" }
```

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `phone` | `string` | 否 | 绑定手机号 |
| `email` | `string` | 否 | 绑定邮箱 |
| `bindType` | `string` | 是 | `PHONE`、`EMAIL`、`BOTH`，大小写不敏感 |

#### 返回值

```json
{ "code": 200, "message": "绑定成功", "data": null }
```

---

### 4.4 获取当前用户信息

- 请求方式：`GET /api/user/info`
- 是否鉴权：是

#### 返回值

```json
{
  "code": 200, "message": "success",
  "data": { "id": "123456789", "role": 1, "username": "test_user", ... }
}
```

---

### 4.5 修改用户名

- 请求方式：`PATCH /api/user/username?username=new_name`
- 是否鉴权：是

#### 返回值：更新后的 UserVO

---

### 4.6 修改性别

- 请求方式：`PATCH /api/user/sex?sex=1`
- 是否鉴权：是

| 参数 | 类型 | 说明 |
|---|---|---|
| `sex` | `int` | `0` 男，`1` 女，`2` 未知 |

#### 返回值：更新后的 UserVO

---

### 4.7 上传头像

- 请求方式：`POST /api/user/avatar`
- 是否鉴权：是
- `Content-Type`：`multipart/form-data`

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `file` | `File` | 是 | 图片文件，`image/*`，最大 5MB |

#### 返回值

```json
{ "code": 200, "message": "头像更新成功", "data": null }
```

#### 业务错误码

| 错误码 | 说明 |
|---|---|
| `40050` | 头像文件不能为空 |
| `40051` | 只能上传图片文件 |
| `40052` | 图片大小不能超过 5MB |

---

### 4.8 修改密码

- 请求方式：`PATCH /api/user/password`
- 是否鉴权：是

#### 请求参数

```json
{ "oldPassword": "123456", "newPassword": "newpass", "confirmPassword": "newpass" }
```

#### 返回值

```json
{ "code": 200, "message": "success", "data": null }
```

---

### 4.9 用户登出

- 请求方式：`POST /api/user/logout`
- 是否鉴权：是

#### 返回值

```json
{ "code": 200, "message": "success", "data": null }
```

> token 会被加入 Redis 黑名单，立即失效

## 6. 病虫害识别模块接口

### 6.1 病虫害识别

- 请求方式：`POST /api/detect`
- 是否鉴权：是

#### 请求参数

```json
{ "imageBase64": "/9j/4AAQSkZJRgABAQ..." }
```

> 不带 `data:image/...;base64,` 前缀

#### 返回值

```json
{
  "code": 200, "message": "success",
  "data": {
    "id": "1743840000000", "userId": "123456789",
    "imageName": "unknown", "topLabel": "brown_spot",
    "confidence": 0.98, "status": 1, "createdTime": 1743840000000
  }
}
```

> `status`：-1=未识别，0=低置信度（<0.7），1=高置信度（≥0.7）

#### 说明

- 后端将图片保存到 `PestDetectionSystem/pest-images/`
- 调用 Flask 模型服务 `/predict`，自动补 `data:image/jpeg;base64,` 前缀
- 低置信度记录仍返回但不出现在历史记录中

---

### 6.2 获取识别记录

- 请求方式：`GET /api/detect/records`
- 是否鉴权：是

#### 返回值

```json
{
  "code": 200, "message": "success",
  "data": [
    { "id": "1743840000000", "userId": "123456789", "imageName": "rice_leaf.jpg",
      "topLabel": "brown_spot", "confidence": 0.98, "status": 1, "createdTime": 1743840000000 }
  ]
}
```

> 仅返回 `status=1` 的记录，按 `createdTime` 倒序

## 8. 对话模块接口

### 8.1 创建会话

- 请求方式：`POST /api/chat/session`
- 是否鉴权：是

#### 请求参数

```json
{ "title": "水稻病害咨询", "scene": "expert" }
```

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `title` | `string` | 否 | 空时默认 `新对话` |
| `scene` | `string` | 否 | 空时默认 `expert` |

#### 返回值

```json
{
  "code": 200, "message": "success",
  "data": {
    "id": "1", "title": "水稻病害咨询", "scene": "expert",
    "messageCount": 0, "lastMessageAt": 1743840000000,
    "createdTime": 1743840000000, "updatedTime": 1743840000000
  }
}
```

---

### 8.2 获取会话列表

- 请求方式：`GET /api/chat/sessions`
- 是否鉴权：是

#### 返回值：`List<ChatSessionVO>`，按 `lastMessageAt` 倒序

---

### 8.3 获取消息历史

- 请求方式：`GET /api/chat/session/{sessionId}/messages`
- 是否鉴权：是

#### 返回值

```json
{
  "code": 200, "message": "success",
  "data": [
    { "id": "1", "sessionId": "1", "detectionId": null, "role": "user",
      "content": "水稻叶片发黄是什么原因？", "model": null,
      "promptTokens": 0, "completionTokens": 0, "totalTokens": 0, "createdTime": 1743840000000 },
    { "id": "2", "sessionId": "1", "detectionId": null, "role": "assistant",
      "content": "可能原因有缺氮、病害或水分问题。", "model": "MiniMax-M2.7",
      "promptTokens": 120, "completionTokens": 60, "totalTokens": 180, "createdTime": 1743840005000 }
  ]
}
```

---

### 8.4 发送消息（普通）

- 请求方式：`POST /api/chat/send`
- 是否鉴权：是

#### 请求参数

```json
{ "sessionId": "1", "message": "水稻叶片发黄是什么原因？", "detectionId": null }
```

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `sessionId` | `string` | 是 | 必须是当前用户已存在的会话 |
| `message` | `string` | 是 | 消息内容 |
| `detectionId` | `string` | 否 | 关联识别记录 ID（当前服务逻辑未使用） |

#### 返回值

```json
{
  "code": 200, "message": "success",
  "data": { "sessionId": "1", "answer": "可能原因有缺氮、病害或水分问题。",
    "promptTokens": 120, "completionTokens": 60, "totalTokens": 180 }
}
```

#### 说明

- `sessionId` 不能为空，不会自动创建会话
- 首轮对话成功后，若会话无消息，自动用首条消息前 20 字生成标题
- 携带最近 20 条成功消息作为上下文
- 每用户每日默认最多 50 次，可通过 `chat.quota.daily-max-requests` 配置

---

### 8.5 发送消息（SSE 流式）

- 请求方式：`POST /api/chat/send/stream`
- 是否鉴权：是
- 返回类型：`text/event-stream`（SseEmitter）

#### 请求参数：同 8.4

#### 说明

- 以 SSE 格式逐块推送模型回复
- 前端使用 `fetch` + SSE 解析接收

---

### 8.6 删除会话

- 请求方式：`DELETE /api/chat/session/{sessionId}`
- 是否鉴权：是

#### 返回值

```json
{ "code": 200, "message": "删除成功", "data": null }
```

> 逻辑删除，`deleted` 字段置为 `1`

---

### 8.7 获取对话额度

- 请求方式：`GET /api/chat/quota`
- 是否鉴权：是

#### 返回值

```json
{
  "code": 200, "message": "success",
  "data": { "quotaDate": "2026-04-18", "requestCount": 15, "inputTokens": 3200, "outputTokens": 1800 }
}
```

> 返回当日统计值；当天无记录时返回 0 值


## 11. 与旧版文档的差异说明

1. 新增 `/user/password`（修改密码）和 `/user/logout`（用户登出）接口
2. 新增 `/admin/info`、`/admin/password`、`/admin/logout` 接口
3. 新增管理员用户管理接口：禁用、启用、删除、修改角色（`/admin/users/{userId}/**`）
4. 新增 `/admin/stats` 系统统计接口
5. 新增 `/chat/send/stream` SSE 流式对话接口
6. `PestDetectionVO` 新增 `status` 字段（-1/0/1）和 `confidence` 字段
7. `ChatMessageVO` 新增 `detectionId` 字段
8. `UserVO` 的 `status` 语义更正：0=正常，1=锁定（原文档描述有误）
9. 所有 ID 字段实际类型为 `String`（雪花 ID 序列化为字符串）
10. token 登出后加入 Redis 黑名单立即失效
